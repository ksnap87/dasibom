**[작업 3] ChatRoomScreen 버그 수정**

버그 1 — Broadcast 채널 레퍼런스 오류:
```tsx
// 기존 (잘못됨): 새 채널 인스턴스 생성 → broadcast 미전달
supabase.channel(`chat_${match_id}`).send({ ... });

// 수정: 구독된 채널을 channelRef로 재사용
channelRef.current?.send({ ... });
```

버그 2 — 읽음 숫자 "1"이 안 사라지는 문제:
- 수신자가 메시지 읽을 때 `read_receipt` broadcast 발신자에게 전송
- 발신자는 수신 시 로컬 메시지 상태의 `read_at` 업데이트 → "1" 사라짐

**[작업 4] TextDecoder 폴리필 추가**
- 문제: React Native JSC 런타임에 `TextDecoder` 내장 없음 → Supabase Realtime WebSocket 오작동
- 오류: `ReferenceError: Property 'TextDecoder' doesn't exist`
- 해결: `text-encoding` 패키지 설치 + `index.js` 최상단에 글로벌 폴리필 추가
```javascript
import { TextDecoder, TextEncoder } from 'text-encoding';
if (typeof global.TextDecoder === 'undefined') global.TextDecoder = TextDecoder;
if (typeof global.TextEncoder === 'undefined') global.TextEncoder = TextEncoder;
```

| **Node.js / Express** | 앱의 두뇌 역할을 하는 서버. 데이터 저장/조회를 처리함 |
| **Supabase** | 클라우드 데이터베이스 + 로그인 서비스. Firebase의 오픈소스 버전 |
| **npm install** | 필요한 라이브러리(부품들)를 인터넷에서 다운받는 명령어 |
| **API 키** | 내 Supabase 데이터베이스에 접근할 수 있는 비밀번호 |
| **스키마(Schema)** | 데이터베이스 테이블 구조 설계도 |
| **RLS (Row Level Security)** | "각 유저는 자기 데이터만 볼 수 있다"는 보안 규칙 |
| **에뮬레이터** | 컴퓨터 안에서 실행되는 가상 Android 폰 |
| **TypeScript** | JavaScript에 타입 검사를 추가한 언어. 오류를 미리 잡아줌 |
| **anon key** | 앱(클라이언트)에서 Supabase에 접근할 때 쓰는 공개 키 |
| **service_role key** | 서버에서만 쓰는 관리자 키. 절대 앱 코드에 넣으면 안 됨 |
