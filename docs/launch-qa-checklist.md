## 시나리오 1 · 신규 가입부터 첫 매칭까지 (해피 패스)

**목표**: 처음 다운받은 사용자가 막힘 없이 매칭·채팅까지 도달하는지

- [ ] 앱 첫 실행 시 스플래시 → Welcome/Auth 화면 전환 3초 내
- [ ] Auth 화면에서 이용약관/개인정보처리방침 링크 두 개가 보이고, 탭하면 브라우저로 각각 200 응답 URL 열림
- [ ] "카카오로 시작하기" → 카카오톡 계정 선택 → 동의 → 다시봄으로 복귀 → 닉네임 설문 1/8 화면 진입
- [ ] 1/8 화면에서 이름·출생연도·성별·거주지역 입력
  - [ ] 출생연도에 **1995** 입력 후 다음 → "40세 이상만 이용할 수 있어요" 경고 표시 ✅
  - [ ] 출생연도에 **1970** 입력 후 다음 → 2/8 진행
- [ ] 2/8 ~ 8/8 설문 모두 완료 → 메인 탭(추천) 진입
- [ ] 추천 카드 최소 1장 로드, 큰 글씨·큰 버튼 OK
- [ ] 카드 좋아요 → "보낸 관심에 대기중" 표시
- [ ] **매칭 탭 → 보낸 관심 펼치기 → 방금 누른 사람 표시**
- [ ] 보낸 관심 행 탭 → 상대 공개 프로필 표시, 채팅 버튼 **없음** (매칭 전이라)
- [ ] 두 번째 계정에서 같은 사람에게 좋아요 → 첫 계정에 FCM 푸시 수신
- [ ] 매칭 탭 자동 새로고침(탭 진입 시) → 채팅 목록에 상대 표시
- [ ] 채팅방 첫 진입 시 본인인증(SMS) 요구
- [ ] 전화번호 입력 → SMS 수신 → OTP 6자리 입력 → 인증 성공
- [ ] 채팅 메시지 송신 → 실시간 수신 확인 (2–3초 내)
- [ ] 메시지에 "010-1234-5678" 입력 → 금칙어 차단 메시지 표시

**예상 소요**: 15분 / 계정 2개 필요

## 시나리오 2 · 크레딧 결제 (Play IAP)

**목표**: 인앱 결제 정상 흐름 + 영수증 서버 검증 정상

- [ ] 크레딧 상점 진입 → 상품 목록 로드
- [ ] 최소 단위 상품 하나 구매 → Google Play 결제 UI 표시
  - **주의**: 내부 테스트 트랙이면 실제 결제 없이 테스트 카드로 처리됨. 프로덕션 트랙은 실결제.
- [ ] 결제 성공 → 크레딧 잔액 +N 반영
- [ ] 추천 탭 → "추가 추천" 버튼 활성, 탭 시 크레딧 -1 차감 + 새 추천 5명 로드
- [ ] 잔액 0 상태에서 "추가 추천" 탭 → 상점 유도 안내

### 결제 엣지 케이스
- [ ] 결제 도중 강제 취소 → 크레딧 미차감 확인
- [ ] 동일 orderId 로 결제 API 수동 재호출 → 서버에서 중복 차단 (`purchase_history` 에 이미 존재)

## 시나리오 3 · 안전 기능

- [ ] 프로필 자기소개에 **"010-5555-6666"** 입력 후 저장 → 400 차단 메시지 한글 표시
- [ ] 프로필 자기소개에 욕설 입력 → 금칙어 차단 메시지
- [ ] 채팅에서 상대방 신고 → 사유 4개 중 하나 선택 → "접수되었습니다" 토스트
- [ ] 채팅에서 상대방 차단 → 매칭 자동 해제, 추천 목록에서 사라짐
- [ ] 차단한 상대를 다시 추천 받는지 확인 — 받으면 안 됨
- [ ] 연락처 동기화 권한 허용 → 저장된 번호로 가입한 지인이 있으면 추천에서 제외
- [ ] 연락처 권한 거부 → 앱 전체는 정상 동작, 지인 제외 기능만 비활성

## 최종 출시 전 Go/No-Go 게이트

출시 승인 조건 — 아래 전부 "YES" 여야만 프로덕션 제출:

- [ ] 시나리오 1–4 전부 통과 (중년 UX 시나리오 5는 "피드백 수집" 목적, 출시 차단은 아님)
- [ ] 테스트 기간 24시간 동안 Crashlytics 에 P0 크래시(자주 발생, 앱 진입 불가) 없음
- [ ] Play Console 의 "앱 콘텐츠" 섹션 모두 초록색 (개인정보처리방침·데이터 보안·콘텐츠 등급·광고·대상 연령대)
- [ ] Railway, Supabase 대시보드에서 최근 1시간 동안 5xx 에러율 < 1%
- [ ] AAB versionCode 가 이전 업로드보다 1 이상 증가

**No-Go 시**: 문제 고치고 새 AAB 빌드 → 내부 테스트 재업로드 → 시나리오 재실행.
